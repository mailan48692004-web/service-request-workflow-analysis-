# Service Request Workflow Analysis: Resolution Time Monitoring and Workflow Performance Review
Built an operational analytics project using R to clean service request data, assess reporting reliability, and analyse turnaround time patterns across request categories. The project focused on recalculating completion metrics, identifying long-tail delays, evaluating category-level performance, and generating recommendations to improve operational monitoring and data quality.

## Project Overview
This project was developed as an operational workflow analytics case study to examine how reliably service request data could be used to monitor completion performance. The dataset contained issues such as missing completion dates, inconsistent completion-time fields, and potential reporting distortions. To address this, I cleaned the data, recalculated the main turnaround metric, and analysed request resolution patterns across categories. The project was designed as a descriptive and diagnostic analysis to support workflow monitoring and process improvement, rather than as a predictive model.

## What I Did
Cleaned service request data and assessed the reliability of key completion-related fields
Handled missing and inconsistent values, particularly around completion dates and completion-time measures
Recalculated days-to-complete to create a more reliable operational metric
Analysed turnaround time distribution to identify typical resolution performance and long-tail delays
Compared request categories by volume and completion speed to identify bottlenecks and priority areas
Generated recommendations for performance monitoring and data-entry improvement

## Tools
R, Tidyverse, Data Cleaning, Data Quality Assessment, Exploratory Data Analysis, Descriptive Statistics, Operational Analytics, Data Visualisation, Workflow Analysis, Business Reporting

## Business Value
The project demonstrates how raw operational data can be transformed into more reliable performance information for workflow monitoring. By separating data quality issues from actual process performance, the analysis helps decision-makers identify where delays are concentrated, which categories should be prioritised, and which reporting fields need tighter control. The result is a more credible basis for service-level monitoring and operational improvement.

## Note
This project is a descriptive operational analytics case study based on historical service request records. It is intended to support workflow monitoring, bottleneck identification, and data quality review. It should not be interpreted as a causal model of process delay.
